# Finances Manager

Simple finance tracker with:

- a desktop/web version powered by Flask
- an Android-ready mobile app powered by Capacitor

Both versions handle the same concepts:

- `UYU` (Uruguayan pesos)
- `USD` (US dollars)

The app runs with Flask and uses an Excel file as a local database, so you do not need an additional database engine to get started.

## Features

- Dashboard with account balance summaries
- Expense and fund entry management
- Transaction history
- Monthly movement charts and spending by category
- Manual balance adjustments
- Local persistence in `finance_data.xlsx`

## Technologies

- Python
- Flask
- openpyxl
- Chart.js
- Vite
- Capacitor

## Requirements

- Python 3.10 or higher
- `pip`
- Node.js 20+

## Installation

```bash
pip install flask openpyxl
npm install
```

## Run Desktop Version

```bash
python app.py
```

Then open in your browser:

```text
http://localhost:5000
```

## How Persistence Works

When the app starts, if `finance_data.xlsx` does not exist, it is automatically created with two sheets:

- `config`: account balances
- `transactions`: transaction history

## Main Structure

- `app.py`: Flask backend and embedded frontend template
- `finance_data.xlsx`: data file generated by the application
- `Start.bat`: quick launcher for Windows
- `src/`: mobile app source code
- `android/`: generated Android project
- `scripts/export_mobile_seed.py`: exports Excel data for mobile import

## Notes

- Balances are handled separately for `UYU` and `USD`
- Adjusting a balance from Settings updates the total, but does not create a transaction
- Deleting a transaction automatically reverses the related balance change

## Android App

The mobile app stores data locally on each phone.
That means if you install the APK on two different devices, each one keeps its own finances separately.

### Build the Mobile Web Assets

```bash
npm run build
```

### Export Your Current Excel Data for Mobile Import

```bash
python scripts/export_mobile_seed.py --output clarifi-mobile-seed.json
```

Then, inside the mobile app, go to `Ajustes` and import that JSON file.

## Generate the APK Locally

The Android project is already created in `android/`.
To build the APK on Windows you need Android Studio or the Android SDK installed.

1. Install Android Studio.
2. Make sure the SDK is available, usually at:

```text
C:\Users\<YOUR_USER>\AppData\Local\Android\Sdk
```

3. Create `android/local.properties` with:

```text
sdk.dir=C:\\Users\\<YOUR_USER>\\AppData\\Local\\Android\\Sdk
```

Or set the `ANDROID_HOME` environment variable to that same path.

4. Build the debug APK:

```bash
npm run apk:debug
```

If everything is configured correctly, the APK will be generated at:

```text
android\app\build\outputs\apk\debug\app-debug.apk
```

## Generate the APK with GitHub Actions

If your local machine does not have the Android SDK configured, this repo also includes a GitHub Actions workflow that builds the debug APK in the cloud.

Workflow file:

- `.github/workflows/android-apk.yml`

After pushing the repo to GitHub, you can run the workflow manually from the `Actions` tab and download the generated APK artifact.
